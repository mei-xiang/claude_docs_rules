# 任务跟踪流程

项目使用 `tasks.md` 进行任务跟踪和断点续传。

## 工作流程

1. **初始化/更新清单**
   - 在开始工作前，检查项目根目录是否存在 `tasks.md`。
   - 如果不存在，先创建它，并将任务拆解为详细的步骤列表（使用 Markdown 复选框格式 `- [ ]`）。
   - 如果已存在，先读取它，确认当前进度。

2. **执行与更新**
   - **只做下一步**：不要试图一次性完成所有任务。
   - **完成后打勾**：每完成一个步骤，必须立即修改 `tasks.md` 文件，将对应的 `- [ ]` 改为 `- [x]`，并简要记录完成结果或遇到的关键问题。
   - **记录上下文**：在 `tasks.md` 的底部，可以维护一个"当前上下文"区域，记录关键变量、文件路径或临时决策，以便下次恢复。

3. **断点恢复**
   - 如果用户重新开启会话，或者指令是"继续"，必须首先读取 `tasks.md`。
   - 找到第一个未打勾的任务，询问用户是否从该处继续。

## 文件格式示例

```markdown
### 任务进度
- [x] 1. 分析数据库结构
- [ ] 2. 编写 User Entity
- [ ] 3. 实现 UserService
### 当前上下文
- 正在处理模块：User Auth
- 关键文件：src/main/java/com/example/User.java
```
